# Scenario matrix — the coverage ledger

The spine of the audit. Every row is a workflow invariant or user scenario the plugin must uphold. Phase 2 probes **close** rows by filling `actual` / `evidence` / `runs` / `environment` and flipping `status` to `PASS`, `FAIL`, or `NOT-RUN:<reason>`. `check_matrix.py` refuses synthesis (Task 12) while any row is `PENDING` or any required cell is blank — so the audit cannot finish "green" with probes silently skipped.

**Columns:** `id` · `scenario` · `surfaces` (owning files) · `task` (which Phase-2 task closes it) · `fixture` · `command` (probe sketch) · `oracle` (the check that decides pass/fail) · `expected` (the observable) · `actual` (probe fills) · `evidence` (probe fills, a path) · `runs` (probe fills) · `environment` (probe fills) · `status`.

**Gate:** `python3 docs/evaluation/checkup/check_matrix.py docs/evaluation/checkup/scenario-matrix.md` — exit 0 only when every row is terminal. At authoring time it correctly reports INCOMPLETE (all rows PENDING).

Environment note: all reads/probes run against the `60eaede` worktree; the `environment` cell records the run environment (e.g. `mac/CC2.1.x/clean-config`).
