## 5. Open questions

1. Is `.draft-vN.md` meant to be mandatory before every new `vN.md` write? The current `/plan` text makes it optional. If it is mandatory, document and test that invariant. If it is not, the timeout recovery needs mechanics beyond a message rewrite.

2. Should "pending draft" mean every `.draft-vN.md`, or only drafts without a valid matching ticket? The guard’s behavior after interruption depends on this definition.

3. Should `--allow-single` cover all legitimate single-entry batches, including resumed and final partial `/adopt` batches, or only projects that began with one component?

4. Should outer `subprocess.TimeoutExpired` receive a different denial from the board’s normal idle timeout?
